# 启动台设计说明

## 产品定位

`launch-box` 是一个 macOS 启动台应用，用户可见名称为 `启动台`。

它希望找回旧版 macOS 启动台那种直接、快速、可拖拽整理的体验，同时加入更实用的组织能力：即时搜索、自定义分类、收藏、最近使用、文件夹、隐藏应用和拖拽排序。

应用常驻菜单栏，也可以通过全局快捷键唤起。主要界面是一个覆盖当前桌面的全屏半透明面板。

## 首个版本范围

- 从 `/Applications`、`/System/Applications` 和 `~/Applications` 扫描应用。
- 用大图标网格展示应用和名称。
- 启动台打开后可以立即搜索。
- 支持点击应用或按回车打开当前选中的应用。
- 提供内置分区：全部、收藏、最近、未分类、隐藏。
- 支持创建、重命名、删除和手动排序自定义分类。
- 支持一个应用加入多个分类。
- 支持在分类内创建文件夹，并把应用拖入文件夹。
- 支持在当前分类内拖拽排序应用和文件夹。
- 用户组织数据持久化到本机应用支持目录下的 JSON 文件。
- 提供菜单栏入口，支持显示/隐藏、重新扫描、设置和退出。
- 提供可配置全局快捷键，默认是 `Option + Space`。

## 架构

应用是一个 SwiftUI macOS 应用，使用 Swift 包管理器组织代码。

- `LaunchBoxApp`：应用入口、菜单栏、设置窗口和覆盖窗口协调。
- `OverlayWindowController`：管理无边框全屏覆盖窗口，并显示在当前屏幕上。
- `AppScanner`：扫描 `.app` 包，提取名称、包标识符、路径和图标相关信息。
- `LaunchStore`：管理应用、分类、文件夹、搜索、收藏、最近使用和当前选择状态。
- `LibraryPersistence`：负责用户分类和布局数据的 JSON 读写。
- `HotKeyManager`：注册全局快捷键。
- `AppLauncher`：通过 `NSWorkspace` 打开应用或在 Finder 中显示应用。
- SwiftUI 视图：覆盖面板、搜索栏、分类条、应用网格、文件夹面板、右键菜单和设置页。

## 数据模型

`LaunchApp` 表示扫描到的应用。稳定身份优先使用包标识符，没有包标识符时使用路径。

`LaunchCategory` 表示用户可见的分类，内部保存有序项目列表。

`LaunchFolder` 表示分类内的文件夹，保存文件夹名称和有序应用列表。文件夹只在分类内展示。

`LaunchLibrary` 保存收藏、最近使用、分类、文件夹内容、隐藏应用和快捷键设置。

## 交互设计

启动台默认打开“全部”分区。顶部显示搜索框和分类条，主体是应用网格。

搜索始终可用。输入关键词会实时过滤应用；按 `Esc` 会先清空搜索，再关闭启动台；按回车会打开当前选中的应用。

拖拽交互分为三类：

- 在网格中拖拽应用或文件夹调整顺序。
- 在分类中把应用拖到另一个应用上创建文件夹，或拖到已有文件夹中加入。
- 把应用拖到顶部的收藏、分类或隐藏入口，快速调整组织方式。

右键菜单提供收藏、加入/移出分类、隐藏、在 Finder 中显示等操作。

## 错误处理

扫描某个目录失败时，应用应继续扫描其他目录，并保留非阻塞提示。

持久化 JSON 损坏时，应用会备份损坏文件，然后使用空配置启动，避免用户卡在启动阶段。

如果全局快捷键注册失败，应用仍然可以通过菜单栏入口使用，并在设置页提示用户重新配置快捷键。

## 验证

- 单元测试覆盖配置读写、应用身份、分类成员关系、文件夹操作、搜索过滤和快捷键分类。
- 手动验证应覆盖：启动应用、打开启动台、搜索、打开应用、创建分类、拖拽应用、创建文件夹、重新扫描、重启后确认数据仍然存在。
